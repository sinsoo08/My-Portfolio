OVERVIEW

어떤 프로젝트인가요

TEAM 아토는 2020년부터 활동해 온 마인크래프트 스토리 어드벤처 맵 제작팀입니다. 그동안 만든 맵을 배포하고 팀을 소개할 공간이 필요해서, 공식 웹사이트를 기획부터 직접 만들고 있어요.

맵 다운로드, FAQ, 팀 소개, 후원 연동까지 한 사이트 안에서 처리할 수 있도록 구성을 잡았습니다.

MY ROLE

내가 한 일

기획, UI/UX 설계, 웹 디자인, 프론트엔드, 백엔드까지 전체 과정을 혼자 담당하고 있어요. 아이디어를 디자인으로 옮기고, 그걸 다시 코드로 구현하는 것까지 한 사람이 책임지는 구조입니다.

완성 이후 운영과 유지보수도 제가 직접 맡을 예정이에요. 처음부터 끝까지, 그리고 그 이후까지 전부 혼자 이어가는 프로젝트입니다.

HTML CSS JavaScript Git GitHub

DESIGN

디자인 방향

색은 파란색·흰색·검은색을 기본 팔레트로 잡았고, 한글 본문에는 Pretendard를, 픽셀 느낌을 살리고 싶은 포인트 영역에는 Press Start 2P를 쓰고 있어요.

배지, 통계 숫자, 장식적인 구분선처럼 '그럴듯해 보이지만 실제로는 정보를 더해주지 않는' 요소는 최대한 빼고, 필요한 정보를 담백하게 전달하는 쪽으로 방향을 잡았습니다.

CHALLENGE

힘들었던 점

모든 걸 혼자 부담해야 한다는 게 생각보다 무거웠어요. 기획도, 디자인도, 프론트도, 백엔드도 판단과 결정이 전부 저한테 집중되다 보니 어느 하나가 막히면 전체 흐름이 멈추는 느낌이었습니다. 팀 프로젝트였다면 누군가와 나눌 수 있었을 고민들을 혼자 끌어안고 가야 했어요.

그리고 생각보다 시간이 촉박했어요. 의뢰를 받은 프로젝트다 보니 막연히 천천히 할 수 있는 게 아니었고, 완성 시점을 의식하면서 작업하는 게 내내 약간의 압박으로 남아 있었습니다.

Claude를 적극적으로 활용했는데, 생각보다 내 의도를 전달하는 게 어려웠어요. 원하는 결과물이 머릿속에 있어도 그걸 프롬프트로 정확하게 표현하지 못하면 엉뚱한 방향의 코드가 나왔고, 나온 소스코드가 만족스럽지 않은 경우도 많았습니다.

RESOLUTION

해결 방법

한꺼번에 다 하려다 보면 부담이 커져서, 목표를 아주 작게 쪼개는 방식을 택했어요. 오늘은 이 부분만 끝낸다고 정하고, 다음 날은 거기서 이어지는 다음 부분을 하는 식으로 매일 조금씩 앞으로 나아갔습니다. 전체를 보면 막막해도 오늘 할 것만 보면 할 수 있다는 감각이 생겼어요.

Claude가 주는 소스코드를 그대로 쓰는 게 아니라, 결과물을 보면서 마음에 안 드는 부분을 제가 직접 손보는 방식으로 작업했어요. Claude는 큰 틀을 잡아주는 도구로 쓰고, 세부 조정은 직접 하는 구조로 역할을 나누니까 훨씬 만족스러운 결과가 나왔습니다.

의뢰한 팀원과는 매일 소통하면서 진행했어요. 작업한 내용을 보여주고, 피드백을 받고, 수정하는 흐름을 매일 반복했습니다. 혼자 만드는 프로젝트지만 의사결정의 많은 부분을 대화로 풀어나갈 수 있었고, 그게 혼자라는 부담을 줄여줬어요.

STATUS

진행 상황

현재 진행 중인 프로젝트라, 기능과 디자인이 계속 조금씩 바뀌고 있어요. 완성되면 TEAM 아토가 만든 맵을 실제로 받아볼 수 있는 통로가 될 예정입니다.
