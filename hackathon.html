OVERVIEW

어떤 프로젝트였나요

2025 제4회 순천향 SW 해커톤은 아두이노를 활용해 사회문제를 해결하는 시스템을 직접 구현하는 대회였어요. 과학기술정보통신부와 순천향대학교가 주최한 행사로, 제한된 시간 안에 아이디어를 실제로 동작하는 형태까지 완성해야 했습니다.

MY ROLE

내가 한 일

기획 단계부터 참여해 해결할 사회문제를 정의하고 시스템 구조를 설계했어요. 개발 파트에서는 Python으로 아두이노와 연동되는 데이터 처리 로직을 직접 구현했고, 시리얼 통신으로 센서 데이터를 받아 조건에 따라 동작을 제어하는 코드를 작성했습니다.

Python Arduino Serial 통신 시스템 설계

PROCESS

개발 과정

초기 기획에서 나온 기능들을 우선순위에 따라 정리하고, 실제로 구현 가능한 범위를 팀원들과 함께 좁혀나갔어요. Python 코드는 반복 테스트를 거치면서 계속 수정했고, 처음 설계와 다르게 동작하는 부분은 직접 디버깅하면서 고쳐나갔습니다. 제한된 시간 속에서 무엇을 구현하고 무엇을 덜어낼지 계속 판단해야 했어요.

CHALLENGE

힘들었던 점

크게 두 가지가 막혔어요. 하나는 외관 디자인이었고, 하나는 알고리즘 구현이었습니다.

외관 쪽에서는 아두이노와 각종 센서, 부품들을 내부에 넣고 외형을 만드는 과정이 생각보다 훨씬 까다로웠어요. 부품 배치에 따라 외형 구조가 달라지고, 실제로 조립해보면 머릿속으로 그린 모양과 다르게 나오는 경우가 많았습니다.

알고리즘 쪽에서는 센서 데이터를 받아서 조건에 맞게 처리하는 로직이 생각만큼 바로 동작하지 않았어요. 예외 케이스가 계속 새로 나왔고, 고칠수록 다른 부분이 어긋나는 상황이 반복됐습니다.

RESOLUTION

해결 방법

막히는 상황마다 혼자 붙들고 있기보다 팀원과 계속 공유했어요. 외관은 부품 배치를 직접 맞춰보면서 팀원 의견을 바로 반영하는 방식으로 구조를 잡아나갔고, 알고리즘도 논리를 말로 설명하면서 빠뜨린 조건이나 허점을 함께 찾아냈습니다.

꾸준한 소통 덕분에 각자 보지 못했던 부분을 서로 잡아줄 수 있었고, 디자인과 기능 모두 초기 계획보다 완성도 있는 결과물로 마무리할 수 있었어요. 소통이 결국 완성도를 결정한다는 걸 이 프로젝트에서 직접 느꼈습니다.

TAKEAWAY

배운 점

하드웨어와 소프트웨어를 함께 다루는 경험이 처음이라 시리얼 통신 타이밍이나 예외 처리에서 예상치 못한 문제를 많이 마주쳤어요. 에러 메시지를 직접 읽고 원인을 찾아가는 과정에서 디버깅 감각이 생겼고, 코드가 실제 장치를 움직이는 순간의 감각을 처음 느꼈습니다.
